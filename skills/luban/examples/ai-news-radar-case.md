# 实战案例：ai-news-radar 的一次完整打磨

> 对象：[LearnPrompt/ai-news-radar](https://github.com/LearnPrompt/ai-news-radar)（约1k星的开源AI资讯雷达）
> 周期：一个对话内（2026-06-10 夜 → 06-11），从验料到 [v0.7.0 发版](https://github.com/LearnPrompt/ai-news-radar/releases/tag/v0.7.0)
> 产出：4 个 PR（[#11](https://github.com/LearnPrompt/ai-news-radar/pull/11) [#12](https://github.com/LearnPrompt/ai-news-radar/pull/12) [#13](https://github.com/LearnPrompt/ai-news-radar/pull/13) [#14](https://github.com/LearnPrompt/ai-news-radar/pull/14)）、1 个新Skill、1 个回测工具、24 个新测试
> 本文所有数字都可在上述链接中查证。

## 验料

项目本身成立（1k星、298 fork、持续活跃），但有一条裂缝：仓库内置的 Skill 是**维护侧**的（触发条件 "Use when working on..."），普通用户装了它、问"今天AI圈有什么"，根本不触发。消费侧整块缺位。

**结论：好料，需调整定位。**

## 访行

对标了 6+ 个同行（全部带URL核实星数）：aihot（中心化API路线，消费侧体验标杆）、CloudFlare-AI-Insight-Daily（⭐1.6k，强依赖Gemini key）、last30days-skill（⭐38.9k，表达天花板：宣言式README+一行安装+零配置分层）、若干 <50 星的同类（反面教材：没有差异化叙事的 AI news skill 默认无人问津）。

**踩坑与对策**：负责搜索的后台子Agent卡在用户看不见的 WebFetch 权限弹窗上 23 分钟。叫停、捞回已有线索、改用 `curl`/`gh api` 前台补完。→ 这条教训进了 SKILL.md 的"子Agent工具纪律"。

## 定位（交叉洞察）

aihot 赢在"问一句话就有"的零摩擦，命门是中心化服务器；本项目的静态 JSON 躺在 GitHub Pages 上——**它本身就是一个无鉴权、不会下线、可 fork 的开放数据接口**，但没人把它当接口用。

一句话新定位：**不租别人的雷达——fork一个自己的：信源你选，数据归你，零API、零Key、零服务器。**

## 过尺（活体检查立功）

结构尺量出 63/100。真正值钱的发现全部来自**活体**：

| 发现 | 来源 | 文档里看得到吗 |
|---|---|---|
| daily-brief.json 静默停更 **8 天**（Actions 全绿） | 拉线上 JSON 对 `generated_at` | 看不到 |
| Google News base64 URL 随机撞出 "llm"，世界新闻被标"模型发布" | 拉真实条目逐条复核 | 看不到 |
| 移动端 10 张统计卡占满三屏才见内容 | 390px 真实渲染截图 | 看不到 |

停更根因：流水线每次都重新生成三个文件，但工作流的 `git add` 白名单从未包含它们——**绿色的CI会撒谎**。

## 慢刨（信任阶梯实录）

粒度随信任升级，但始终一提交一面、每提交独立验证：

1. **首轮单面**：新增消费侧 `ai-radar` SKILL.md（命令全部用线上真实数据跑通）+ CI白名单修复。
2. **批量授权"三个都做"**：README三通道首屏、radar README+可复现demo GIF（vhs tape入库）、CI提交语义加固（git行为先在/tmp沙盒验证）。
3. **批量授权"五个问题都解决"**：URL只许主机名参与关键词匹配（当日数据回放：**23条假AI翻转，零误伤**）、精选同源衰减（AIbase占比 **15/20→4/20**）、近重复抑制、移动端2列统计卡、分组渲染封顶（首屏 **806→523卡，页高-30%**）。
4. **批量授权"三刀全做"**：宁缺毋滥门槛（715故事→95过门槛→20精选）、同簇全天去重、热点视图（多源聚簇×时间衰减）、离线回测工具。

**事故实录**：一个遗留的后台克隆进程半小时后失败清理，删掉工作目录和两个未推送提交。内容凭上下文重建，零损失，但代价是二十分钟返工。→ "commit即push""长任务不进后台"进了工位纪律。

## 验证资产沉淀

- 一次性的对比脚本固化成 `scripts/backtest_scoring.py`：任意两个 git 版本的评分逻辑在档案上重放。首份报告：**83,725 条回放，URL修复累计清除 327 条假AI，救回 4 条 `A.I.` 风格标题**。
- 立成项目明文规矩：**动评分必须附 ≥14 天回放报告。**

## 停手点实录

打磨者一度想顺手 merge——被拦下，理由：用户那句"五个问题都解决了吧？"是疑问句，不是授权。下一轮用户说"我们来一个merge和release发版"（祈使句），才执行。→ 这条判断细则原文进了强制停手点。
