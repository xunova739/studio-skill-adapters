# 布局规则库（Layout Rules）

每种图类型都有固定的布局规则。画图前先识别图类型，加载对应规则，再填内容。不靠 AI 发挥布局。

---

## 图类型识别

用户描述中出现以下关键词时，对应到图类型：

| 关键词 | 图类型 | 重复单元方向 |
|--------|--------|-------------|
| 流程、步骤、从…到… | 横向流程图 | 横向延伸 |
| 泳道、多角色、SOP、跨部门 | 泳道图 | 纵向叠加（已有模板，见 swimlane-guide.md） |
| 架构、系统、模块、分层 | 架构图 | 纵向分层 |
| 漏斗、转化、筛选 | 漏斗图 | 纵向收窄 |
| 时间轴、里程碑、阶段 | 时间轴图 | 横向延伸 |
| 思维导图、脑图、发散 | 思维导图 | 中心放射 |
| 对比、vs、比较 | 对比图 | 横向并列 |
| 组织架构、汇报关系 | 组织架构图 | 树状展开 |

---

## 核心机制：可重复单元

每种图类型都定义一个**最小可重复单元（Unit）**。AI 从用户描述中提取节点数量 N，按规则重复 N 次，自动计算坐标。

### 通用参数

| 参数 | 含义 | 默认值 |
|------|------|--------|
| `UNIT_W` | 单元宽度 | 按图类型定义 |
| `UNIT_H` | 单元高度 | 按图类型定义 |
| `GAP_X` | 单元间横向间距 | 按图类型定义 |
| `GAP_Y` | 单元间纵向间距 | 按图类型定义 |
| `MARGIN` | 画布边距 | 60 |
| `TITLE_H` | 标题区高度 | 80 |

### 坐标计算公式

**横向排列**（流程图、时间轴）：
```
第 i 个单元的 x = MARGIN + i × (UNIT_W + GAP_X)
第 i 个单元的 y = MARGIN + TITLE_H
```

**纵向排列**（漏斗图、泳道行）：
```
第 i 个单元的 x = MARGIN
第 i 个单元的 y = MARGIN + TITLE_H + i × (UNIT_H + GAP_Y)
```

**放射排列**（思维导图）：
```
角度 = i × (360 / N)
第 i 个单元的 x = CENTER_X + RADIUS × cos(角度)
第 i 个单元的 y = CENTER_Y + RADIUS × sin(角度)
```

## 各图类型布局规则

### 1. 横向流程图

**适用**：线性步骤、工作流、审批流

**可重复单元**：圆角矩形（步骤）+ 右侧箭头
- `UNIT_W`: 180
- `UNIT_H`: 80
- `GAP_X`: 60（箭头占用）
- 箭头从当前单元右边中点 → 下一单元左边中点
- 第一个单元用 `ellipse`（起点），最后一个用 `ellipse`（终点），中间用 `rectangle`

**布局规则**：
- 所有节点 Y 坐标相同（严格水平对齐）
- 判断节点用 `diamond`，从菱形的下角引出"否"分支，右角引出"是"分支
- 单行超过 5 个节点时换行：第 6 个节点回到最左侧，Y += UNIT_H + GAP_Y × 2，用反向 U 型箭头连接
- 每个节点内文字居中，字号 16，最多 2 行（超出则缩写）

**箭头规则**：
- 同行节点：纯水平箭头
- 换行连接：从右端末尾节点垂直向下 → 水平向左到下一行起点（Z 形折线）
- 分支箭头（判断节点）：在箭头中点旁标注条件文字（"是"/"否"）

---

### 2. 架构图

**适用**：系统架构、技术栈、模块分层

**可重复单元**：横向层（Layer），每层包含 1-N 个模块矩形
- `LAYER_H`: 100（每层高度）
- `GAP_Y`: 40（层间距）
- `MODULE_W`: 140（模块宽度）
- `MODULE_GAP`: 20（同层模块间距）

**布局规则**：
- 从上到下分层：用户层 → 接入层 → 业务层 → 数据层
- 每层用虚线矩形框做背景容器，左上角标注层名
- 层内模块水平居中排列
- 层间箭头从上层模块底部中点 → 下层模块顶部中点
- 同层模块间的关系用水平双向箭头

**分层原则**：
- 用户/客户端：最顶层
- API 网关/负载均衡：第二层
- 业务服务/微服务：第三层
- 数据存储/基础设施：最底层
- 横切关注点（监控、日志、安全）：放在右侧竖向条

---

### 3. 漏斗图

**适用**：转化率、筛选流程、招聘管道

**可重复单元**：梯形区域 + 右侧数据标注
- `TOP_W`: 400（最宽处）
- `BOTTOM_W`: 100（最窄处）
- `STEP_H`: 60（每级高度）
- `GAP_Y`: 8（级间距，极小以保持漏斗感）

**布局规则**：
- 第 i 级宽度 = TOP_W - i × ((TOP_W - BOTTOM_W) / (N - 1))
- 每级水平居中（x = CENTER_X - 当前宽度/2）
- 每级左右两侧的梯形边对齐形成收窄效果
- 右侧标注区：每级对应的数字/百分比，与该级垂直居中对齐
- 颜色渐变：从顶部浅色到底部深色（表达"越筛越精"）

**Excalidraw 实现**：
- 每级用 `rectangle`，宽度按公式递减
- 级间用短箭头连接
- 右侧数据用自由浮动文字，x = 漏斗右边 + 40

---

### 4. 时间轴图

**适用**：项目里程碑、阶段规划、历史事件

**可重复单元**：时间节点（圆点 + 上/下标注）
- `NODE_R`: 8（圆点半径）
- `GAP_X`: 160（节点间距）
- `LABEL_OFFSET_Y`: 40（标注文字距轴线的偏移）

**布局规则**：
- 横向主轴线：一条水平直线贯穿所有节点
- 节点圆点在轴线上，等间距排列
- 奇数节点标注在上方，偶数节点标注在下方（避免拥挤）
- 每个标注包含：时间（小字灰色）+ 事件名（正常字号）
- 重要节点用大圆点 + 主题色填充，普通节点用小空心圆

**箭头规则**：
- 时间轴本身不是箭头，是普通直线
- 最右端加一个箭头表示"未来方向"
- 不在时间轴上画额外箭头（避免混乱）

---

### 5. 思维导图

**适用**：概念发散、头脑风暴、知识结构

**可重复单元**：分支（中心 → 一级节点 → 二级节点）
- `CENTER_R`: 40（中心节点半径）
- `L1_DISTANCE`: 200（一级节点距中心距离）
- `L2_DISTANCE`: 120（二级节点距一级节点距离）
- `L2_SPREAD`: 30（二级节点间角度偏移）

**布局规则**：
- 中心节点居画布中央
- 一级节点按 360°/N 均匀分布在中心周围
- 二级节点在一级节点的外侧，以一级节点为圆心、±SPREAD 角度排列
- 连线用曲线（Excalidraw 的 `line` + roundness），不用直线
- 中心节点最大字号 + 最深颜色，一级节点中等，二级节点最小最浅

**配色规则**：
- 每个一级分支一个主题色（从 color-palette.md 取，不超过 6 色）
- 同分支的二级节点用该分支色的浅色版

---

### 6. 对比图

**适用**：方案对比、A vs B、优劣分析

**可重复单元**：对比列（列标题 + N 行属性值）
- `COL_W`: 200（每列宽度）
- `ROW_H`: 50（每行高度）
- `HEADER_H`: 60（标题行高度）
- `GAP_X`: 0（列间无间距，用竖线分隔）

**布局规则**：
- 第一列为属性名列（左对齐，灰色背景），宽度可以比数据列窄
- 后续列为对比对象，标题行带主题色
- 奇偶行交替底色（极浅灰 / 白）提升可读性
- 优劣标记：✅ / ❌ / ⚠️ 或颜色填充（绿 / 红 / 黄）
- 最多 4 列对比对象（第一列属性名 + 4 列 = 5 列总计），超过拆成两张图

---

### 7. 组织架构图

**适用**：团队结构、汇报关系

**可重复单元**：人员卡片（姓名 + 角色）
- `CARD_W`: 140
- `CARD_H`: 60
- `GAP_X`: 40（同级间距）
- `GAP_Y`: 80（层级间距）

**布局规则**：
- 最高层居中，下级层水平居中对齐于上级
- 每个节点的 X = 其所有子节点的 X 中心
- 连线从父节点底部中点 → 垂直向下到子节点层 Y 上方 → 水平到子节点 → 垂直到子节点顶部中点
- 同级节点间的 GAP_X 保持一致
- 叶子节点层决定整张图的总宽度

**注意**：树状结构的坐标需要从叶子节点向上推算，不能从根节点向下。先排好叶子层，再逐层向上计算父节点居中位置。

## 箭头语义分类

不同语义的箭头用不同样式，让读者一眼区分流转类型：

| 语义 | 线型 | 颜色 | 用途 |
|------|------|------|------|
| 主流程 | 实线，箭头 | 默认黑/深灰 | 正常步骤流转 |
| 数据流 | 实线，箭头 | 蓝色 `#1e40af` | 数据传输方向 |
| 触发/事件 | 虚线，箭头 | 橙色 `#c2410c` | 异步触发、事件通知 |
| 反馈/返回 | 虚线，箭头 | 灰色 `#64748b` | 结果返回、反馈循环 |
| 错误/异常 | 虚线，箭头 | 红色 `#dc2626` | 错误处理路径 |
| 跨角色交接 | 虚线，箭头 | 紫色 `#7c3aed` | 跨泳道协作 |

在图例区域标注使用的箭头类型，不要让读者猜。

## 自适应画布尺寸

不要用固定画布，根据内容量动态计算：

```
画布宽度 = MARGIN × 2 + 内容列数 × (UNIT_W + GAP_X) - GAP_X
画布高度 = MARGIN × 2 + TITLE_H + 内容行数 × (UNIT_H + GAP_Y) - GAP_Y
```

Excalidraw 没有画布边界限制，但 `appState.scrolledLeft` 和 `scrolledTop` 需要设为合理初始位置，让用户打开时看到图的中心。
