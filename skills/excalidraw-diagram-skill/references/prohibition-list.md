**这是一个活文档，像 Zettelkasten 卡片盒一样自我维护。**

每条规则有完整生命周期：`草稿 → active → 候选删除 → 已毕业`。
- **触发次数为 0 且超过 30 天**：标记为"候选删除"
- **两条规则来源场景重叠 > 80%**：标记为"候选合并"
- **有测试用例完全覆盖**：标记为"候选毕业"（模型已内化，可降级到 test-cases.md 兜底）

不符合以下三条的规则不加入：① 可执行 ② 防止真实失败 ③ 可抽象到同类场景

## [布局]

❌ **不要省略层级背景色块（layer background）**
   为什么：没有背景分区的图，各层元素视觉上混在一起，读者无法快速识别哪些元素属于同一层。人眼靠「区域归属」建立分组认知（格式塔-共同区域原则），缺失背景色块等于丢失了最重要的分组线索
   替代方案：每一层用大矩形（dashed 边框 + 极浅填充色）包裹所有子元素，形成视觉分区。背景色必须比子元素填充色浅至少 3 级，避免抢夺前景注意力
   来源场景：架构图省略了层级背景，只用自由浮动文字做分隔，结果看起来像一堆散落的卡片
   触发次数：1
   最后触发：2026-06-15
   状态：active
   关联规则：→ [布局-留白重要性]

❌ **不要让所有节点大小相同、间距相等**
   为什么：等大等距的网格没有视觉层级，用户看不出重要/次要之分
   替代方案：主角节点 300×150，用留白（200px+）突出重要元素
   来源场景：多步骤流程图，所有步骤被画成等大矩形
   触发次数：0
   最后触发：-
   状态：active
   关联规则：→ [布局-留白重要性]

❌ **不要把箭头直接穿过其他元素**
   为什么：无法分辨箭头是穿过还是连接到中间元素
   替代方案：给箭头的 `points` 数组加中间折点，绕过障碍物
   来源场景：跨越多个区块的长距离连接
   触发次数：0
   最后触发：-
   状态：active
   关联规则：→ [JSON-boundElements]

❌ **不要对所有节点用同一种颜色**
   为什么：颜色没有传达任何语义信息，等于没有配色
   替代方案：从 color-palette.md 按语义选色
   来源场景：批量生成的流程图，所有节点统一蓝色
   触发次数：0
   最后触发：-

   来源场景：包含分支逻辑的流程图

   最后触发：-

## [JSON]

❌ **不要用英文字符宽度系数计算中文文本的 width**
   为什么：英文字母平均宽度约 0.5-0.6em，但中文/日文/韩文是等宽字符，每字占 ~1.0em。用英文系数会导致 text 元素的 width 严重不足，Excalidraw 渲染时文字被截断或挤压只显示一半
   替代方案：检测文本是否含 CJK 字符（Unicode 一-鿿），中文用 0.9-1.0 系数，纯英文用 0.55，混合内容用 0.75。宁可宽一些也不要窄
   来源场景：Python 脚本生成 JSON 时用 `len(line) * fontSize * 0.55` 计算宽度，中文标题「候选人全流程状态机」实际宽度远大于计算值
   触发次数：1
   最后触发：2026-06-15
   状态：active
   关联规则：无

❌ **不要在 `text` 字段里写 HTML 或 Markdown**
   为什么：Excalidraw 直接渲染文本，标签会显示为原始字符
   替代方案：text 字段只写纯文字，换行用 `\n`
   来源场景：从文档提炼内容时直接复制带格式的文字
   触发次数：0
   最后触发：-
   状态：active
   关联规则：无

❌ **不要忘记同步更新 `boundElements`**
   为什么：箭头绑定是双向的，只写一侧会导致渲染时箭头脱落
   替代方案：添加箭头时同时更新两端元素的 `boundElements`
   来源场景：分段生成大图时跨段箭头绑定
   触发次数：0
   最后触发：-
   状态：active
   关联规则：→ [布局-箭头穿越]
